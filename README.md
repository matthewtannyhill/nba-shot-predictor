# NBA Shot Predictor

Two scikit-learn models trained on public NBA datasets, served from a Next.js frontend with Python serverless functions on Vercel.

**Live:** https://nba-shot-predictor.vercel.app

## What it predicts

- **Shot model** — given pre-shot tracking features (shot clock, touch time, shot distance, defender distance, dribbles), returns the probability the shot goes in.
- **Salary model** — given a player's season stats (age, games, minutes, OWS, DWS, WS), returns a predicted NBA salary.

## History

Originally a 2019 BYU class project (Django + Azure ML Studio). Microsoft retired Azure ML Studio (classic) on August 31, 2024, taking the prediction backend down with it. Rebuilt in 2026 with locally-trained sklearn models, a Next.js frontend, and Python serverless functions. Original repo: https://github.com/matthewtannyhill/nbashots.

## Stack

| Layer | Tech |
|---|---|
| Frontend | Next.js 16, React 19, Tailwind 4 |
| Inference | Python 3.12 serverless functions on Vercel |
| Training | scikit-learn 1.5, joblib, pandas |
| Data | Kaggle (`dansbecker/nba-shot-logs`, `drgilermo/nba-players-stats`, `whitefero/nba-player-salary-19902017`) |

## Honest model performance

| Model | Metric | Value |
|---|---|---|
| Shot | AUC | 0.634 |
| Shot | Accuracy | 61.9% |
| Salary | R² (log) | 0.371 |
| Salary | Median absolute error | ~$940k |

The shot model's AUC ceiling on this feature set is well-documented — defensive scheme, fatigue, and contract-year effects aren't in the data. The salary model treats 1995 dollars and 2017 dollars the same, which they aren't.

## Local development

Requires Python 3.9+, Node 20+, and a free Kaggle account.

```bash
# Python deps
python3 -m venv .venv
source .venv/bin/activate
pip install kagglehub python-dotenv pandas scikit-learn joblib openpyxl

# Kaggle auth: drop your token in .env
echo "KAGGLE_API_TOKEN=KGAT_..." > .env

# Pull the datasets
python3 scripts/download_data.py

# Train both models (writes to models/)
python3 scripts/train_shot_model.py
python3 scripts/train_agent_model.py

# Frontend
npm install
npm run dev
```

The `models/` pickles are committed so the Vercel functions can find them at runtime; `data/` and `.venv/` are gitignored.

## Layout

```
src/app/         Next.js routes (home, /shot, /salary, /about)
api/             Python serverless functions
models/          Trained .pkl files (committed; ~90MB total)
scripts/         Training and dataset download scripts
data/            Raw CSVs (gitignored, regenerated by scripts/download_data.py)
```

## Caveats and known limits

- Both function bundles exceed Vercel's 250MB limit due to sklearn's footprint, so Vercel falls back to runtime dependency installation. First request after cold start can take 5-15 seconds.
- Switching to LightGBM or ONNX would shrink each model 50x and remove the cold-start delay. Future work.
- Salary predictions aren't inflation-adjusted.
